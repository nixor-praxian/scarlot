MVP -- SAFETY + MEMORY
Safety Coverage
Shared blacklist network + personal client intelligence. Phone number as unique key. Auto-populated contact records on first message. Passive background checks when a new number arrives. GABRIELLE's scraper proves the need. Near-zero reporting friction required.
P5 P3 P2 P6 #blacklist #crm
Evidence: Gold (all 8 interviews). Design constraint: reporting friction must be near-zero (mood-dependent behavior = data loss).
PHASE 1 -- SWITCHBOARD
Framework & Interactions
Conversational interface in WhatsApp. Pre-recorded message triage (GABRIELLE's pattern). Template responses for rates, services, location. Semi-automatic, never full-auto. "C'est important d'etre la" -- the human must stay present.
P8 P4 #automation #templates
A7 boundary clarified: passive checks YES, AI conversation NO. Users want automatic background checks, reject AI speaking for them.
PHASE 1 -- MEMORY + BOOKING
Professional Organization
Client intelligence profiles, booking management, availability broadcasting. MARYLINE's emoji system, GABRIELLE's Notion DB, CAROLL's contact renaming -- all replaced by auto-populated records. Natural language availability declaration.
P2 P4 #crm #scheduling
Evidence: Gold + Bronze (P2 Gold, booking Bronze). Minimum fields: number, first contact date, source channel, risk flag, free-text note.
PHASE 2 -- PAYMENTS + COMPLIANCE
Transactions
Income tracking, session logging. GABRIELLE built a bank notification parser. Tracks seasonality, revenue per customer, costs, taxes. Wants Mint-like aggregator. Cash-dominant business needs lightweight session logging.
P16 #financial #tracking

PHASE 2-3 -- REPUTATION
Collective Blacklist
Graduated reporting model: personal label for first offense, community report for repeated offenses. Credit bureau model + NUM precedent. Existing infrastructure: Telegram groups, WhatsApp communities, Projet Jasmine, CheckClient.
P5 P7 #collective #legal
Legally unresolved. A1 supported behaviorally but design constraint: contribution AND consumption friction are both barriers.
PHASE 1 -- IDENTITY
Resources & Identity

P17 #identity #resources
Co-founder insight, reinforced by GABRIELLE's Gold signal. Platform must allow identity self-definition.
PHASE 3 -- COMPLIANCE
Analysis & Insights
GABRIELLE tracks seasonality, revenue per customer. Behavioral data from the platform enables business intelligence. Pattern detection, trend analysis, predictive insights. But only after trust is established through core value delivery.
P16 #analytics #intelligence
Phase 3 only. Requires critical mass of usage data. GABRIELLE is the only user currently tracking at this level.

Validated Design Constraints

C1
Near-zero reporting friction

Data sovereignty guarantee
GABRIELLE left Notion over account suspension risk. TDS require visible control over their data. On-device where possible, Swiss hosting for shared features.
